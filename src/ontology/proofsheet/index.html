Details about a selected ontology or package are shown here. All other tabs focus on the selected resource.

Browsing, searching and shopping for terms can only involve one version of an ontology or package at a time.

If you login, your persistent list of downloadable packages will be listed in the "My Packages" section.

A package is a collection of ontology term and entity identifiers and filters generated from a shopping list. A package enables generation of an ontology-driven entity specification file (as an Excel spreadsheet or in JSON format) which indicates term names, synonyms, definitions, and categorical and numeric variable details. A specification file can be periodically regenerated from a package so that new ontology updates can be incorporated into it. A specification can be used by software and data integration developers to facilitate data entry, display and exchange.

A package cannot contain terms from two versions of an ontology, but it can contain more than one ontology.

To download a JSON or excel specification from a package, click on its name, and then view the "Data Specification" tab where it will be regenerated.

One can edit a package to adjust its name or description. Upgrading a package involves changing its ontology version number(s). A differential list of changes is provided so that decisions can be made about what new terms to accept or filter out.
