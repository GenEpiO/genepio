# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

GenEpiO (Genomic Epidemiology Ontology) is an OBOFoundry application ontology managed with the [Ontology Development Kit (ODK) v1.6.1](https://github.com/INCATools/ontology-development-kit). All builds run inside a Docker container via `run.sh`.

## Build Commands

All commands must be run from `src/ontology/`. They require Docker to be running.

```bash
# Full build (generates all OWL/OBO release artifacts)
sh run.sh make

# Run all QC checks (tests, SPARQL validation, ROBOT reports)
sh run.sh make test

# Fast tests — skips imports, patterns, mirrors (matches CI behavior)
sh run.sh make test IMP=false PAT=false MIR=false

# Refresh all imported ontology modules
sh run.sh make all_imports

# Refresh a single import (e.g., general)
sh run.sh make refresh-general

# Run only SPARQL validation queries
sh run.sh make sparql_test

# Run only the ELK reasoner consistency check
sh run.sh make reason_test

# Update ODK version and regenerate managed config
sh run.sh make update_repo
```

### Debugging OWL errors

If `make` or `robot` raises `java.lang.IllegalArgumentException: URI is not absolute`, validate the OWL/XML form of the edit file using Apache Jena's `riot`:

```bash
# Install on Mac
brew install jena

# In Protégé, save a copy as genepio-validate.owl (OWL/XML syntax), then:
riot -v --validate genepio-validate.owl
```

After a successful build, verify ROBOT can read the result:

```bash
robot convert -i genepio-merged.owl -o genepio.json
```

## Repository Architecture

### Source files (edit these)

- `src/ontology/genepio-edit.ofn` — Main edit file in OFN (OWL Functional Notation). This is the primary file curators modify in Protégé.
- `src/ontology/genepio-odk.yaml` — ODK project configuration (edit format, imports, components, CI settings).
- `src/ontology/genepio.Makefile` — **Custom make rules** (e.g., ontofox-based import fetching). Add custom build logic here.
- `src/ontology/components/*.ofn` — Modular component files imported into the main edit file:
  - `specification_field.ofn`, `specification_enum.ofn`, `specification_mint.ofn` — Data standard term definitions
  - `gazetteer.ofn`, `gaz_insdc_mapping.ofn` — Geographic/INSDC mappings
  - `deprecation.ofn`, `hardcoded.ofn`, `temporary_mint.ofn`
  - `specification.owl` — Generated from ROBOT templates; declared in `genepio-odk.yaml` under `components: products:` as `- filename: specification.owl`, which causes ODK to set it up as an imported OWL file in `genepio-edit.ofn` (do not edit directly)
- `src/templates/` — ROBOT template TSV files; `specification.owl` is generated from these.
- `src/sparql/` — SPARQL queries used for validation (`*-violation.sparql`) and reporting (`edges.sparql`, `synonyms.sparql`, etc.).

### Generated / do not edit

- `src/ontology/Makefile` — Auto-generated by ODK from `genepio-odk.yaml`. **Do not edit.** Regenerate it with `sh run.sh make update_repo` after changing the ODK config.
- `imports/*_import.owl` — Extracted subsets from external ontologies.
- `genepio-base.owl`, `genepio-full.owl`, `genepio-simple.owl`, `genepio.owl`, `genepio.obo` — Release artifacts built by `make`.

### Import management

Custom imports are fetched via the [ontofox](https://ontofox.hegroup.org/) service using specification files in `src/ontology/imports/*_ontofox.txt`. The five custom import groups are: `ro`, `obi`, `chebi`, `ncbitaxon`, and `general`.

### Key configuration

- `src/ontology/catalog-v001.xml` — OASIS XML catalog mapping semantic URIs to local file paths. Updated automatically when imports change.
- `src/ontology/genepio-idranges.owl` — ID range assignments per curator.
- `project.yaml` — Legacy project metadata (primary release: `full`).

## CI/CD

GitHub Actions workflows in `.github/workflows/`:

- **`qc.yml`** — Runs on push/PR using `obolibrary/odkfull:v1.6.1`; executes `make test IMP=false PAT=false MIR=false` with 6 GB Java heap.
- **`docs.yml`** — Deploys MkDocs documentation to GitHub Pages.

## Memory / Java settings

Default: `ROBOT_JAVA_ARGS=-Xmx2048m`. For large imports or full builds:

```bash
sh run.sh make ROBOT_ENV='ROBOT_JAVA_ARGS=-Xmx6G'
```
